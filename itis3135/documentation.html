1. Overview

The Jolly Dolphin Study Club is a friendly space for students to practice web development skills. We focus on building projects, improving study habits, and supporting each other during assignments.

  • We aim for progress, not perfection.
  • We help each other without copying work.
  • We keep everything organized so it’s easy to submit.

2. Meeting Rules

Meetings are short and focused. Members can join for one sprint or stay longer. Respect matters more than speed.

  • Be respectful and patient.
  • Ask clear questions (what you tried, what error you got).
  • No copying a classmate’s full solution.

3. Study Sprints

A study sprint is a timed focus session. We normally use a 25/5 cycle: 25 minutes work, then a 5 minute break.

Example sprint plan:

  1. Pick one small task (example: “finish nav bar”).
  2. Work for 25 minutes without distractions.
  3. Take 5 minutes break.
  4. Repeat if needed.

4. Debugging Process

When something breaks, we use a simple “Dolphin Debug” checklist. This helps us fix issues fast and avoid guessing.

  • Read the error message carefully.
  • Check spelling (file names, links, tags).
  • Validate HTML/CSS.
  • Test one change at a time.

Example of a common fix (missing closing tag):

<p>This is a paragraph.
</p>

Example of linking a CSS file correctly:

<link rel="stylesheet" href="styles/default.css">

5. File Structure

We keep our project files in a clean structure so pages can find the CSS and images correctly. 

itis3135/
  index.html
  contract.html
  survey.html
  cards.html
  inventory.html
  documentation.html
  product.html
  styles/
    default.css

Tip: If a page cannot find the CSS, the most common reason is the file path is wrong.

6. Help Etiquette

Getting help is encouraged, but we do it in an ethical way. The goal is to learn, not just “get it done.”

  • Share what you tried first.
  • Ask for explanation, not copy-paste answers.
  • Always credit sources if you used them.
